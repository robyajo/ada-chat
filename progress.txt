- Created `components/OAuthCallback.tsx` — handles Google/Discord OAuth redirect from backend

  - Saves tokens via setTokens() to localStorage
  - Redirects to / on success, shows error on failure
- Refactored `App.tsx` — auth state separated from chat session
  - Detects /oauth/callback path -> renders OAuthCallback
  - On mount, checks localStorage for token -> fetches /api/v1/auth/me for authUser
  - Three render states: OAuthCallback / Chat (when session exists) / Lobby
  - saveSession/loadSession helpers for chat_session localStorage
- Refactored `components/Lobby.tsx` — new flow: login -> room selection -> chat
  - New props: authUser, onAuthSuccess, onLogout (replaces standalone auth logic)
  - Auth form shown only when not authenticated
  - After auth success -> stays on Lobby showing Room Selection (Join/Create buttons)
  - Patuih key setup shown after login if user has no key, then back to Room Selection
  - Logout button in Room Selection view
  - Only calls onEnter() when user actually joins or creates a room
